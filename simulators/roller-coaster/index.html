// Red Team Report — Pre-Delivery Audit
✓ FINDING 1 — FIXED: Loop Test Height Calculation Error
Python version test computed h = 2×R×sin(45°)+R = 24.14m instead of correct R + R×sin(45°) = 17.07m. This caused loop g-force verification to return 3.36g instead of the expected 4.77g. Core calculation logic was correct — only the test setup had the wrong height. HTML version uses correct height formula throughout.
✓ FINDING 2 — DOCUMENTED: PDF Typo in Example 5c
Source PDF states a = 37.018 m/s² for braking deceleration, but the correct math is: 81054 N ÷ 3000 kg = 27.018 m/s² = 2.76g. The PDF value of 37.018 appears to be a typo from an earlier version of the problem with different numbers. This simulator uses the mathematically correct value and notes the discrepancy explicitly on the Work-Energy tab.
✓ FINDING 3 — FIXED: Banked Curve Domain Error
Original code: Math.acos(1/g_felt) crashes when g_felt < 1.0 (domain error). A banked curve cannot produce g_felt < 1 by physics definition (minimum is 1g at θ=0°), but unguarded user input would crash silently. Fixed: HTML version validates g_felt ≥ 1.001 before executing arccos and displays a clear error message.
✓ FINDING 4 — FIXED: Spring Division by Zero
Original code: spring_constant = (mass × 107.8) / x_max crashes on x_max = 0. Fixed: HTML version validates x_max > 0 and enforces minimum input of 0.1m.
⚠ FINDING 5 — KNOWN LIMITATION: Friction Not Modeled
This simulator uses ideal energy conservation (no friction losses). Real coasters lose approximately 5-15% of energy to wheel friction and air resistance. For educational purposes this is standard AP Physics assumption. Students should note that real coaster velocities will be lower than calculated values, requiring more initial height.
⚠ FINDING 6 — KNOWN LIMITATION: Train Length Effects
Physics is calculated at the center of mass of the train. The PDF section on Center of Mass explains that riders at the rear of a train experience higher g-forces over hills than the calculated value. This simulator models a point mass, not a distributed train. For full accuracy, multiply rear-car g-forces by approximately 1.2-1.4× on hill crests.

// PASS 2 — CITATION-BACKED PEER REVIEW
✓ PASS-2 FINDING 1 — FIXED: getGStatus() called 7.99g "SAFE" (green badge)
The function previously returned a green "SAFE" badge for anything between 1g and 7.99g. This is directly contradicted by the source document itself:

• Tony Wayne AP Physics Guide (the reference PDF): "Most roller coasters keep the g's felt under 5g"
• CoasterForce (citing NASA Bioastronautics Data Book 1964): "Positive G-Forces becomes uncomfortable at +5gz — at 5gz humans are at risk of loss of consciousness"
• Coasterpedia: "4G is considered a safe limit for sustained forces"
• Wikipedia / High-g training: "Roller coasters typically do not expose occupants to much more than about 3g"

Fixed: Three-tier system. SAFE (≤5g) | CAUTION (5–6.5g) | BLACKOUT-RISK (6.5–8g) | BLACKOUT (8g+).
✓ PASS-2 FINDING 2 — FIXED: Track Builder brake distance was hardcoded
Line 1313 (original): const dist = Math.abs(seg.h1 - seg.h2) + 5; // rough estimate
For a flat brake section (h1=h2=0), this always used dist=5m regardless of actual brake zone. The braking force output appeared precise to the Newton but was computed from a silent guess. A physics simulator presenting fabricated-looking precision over a hidden estimate is a reliability failure. Fixed: Brake zone length is now a required user input field (default 10m).
✓ PASS-2 FINDING 3 — FIXED: Negative g intermediate zone missing
Previous code treated all negative g between 0 and -3g identically as "NEG-G" warning. But CoasterForce (NASA source): "at -2gz: severe headache, nosebleeds, swelling of eyelids" and "on a coaster a level higher than -1gz would be considered dangerous."
Fixed: Three zones — AIRTIME (0 to -1g) | NEG-G caution (-1g to -2g) | REDOUT-RISK (-2g to -3g) | REDOUT (-3g+).
✓ PASS-2 FINDING 4 — FIXED: Formula notation ambiguous (v²/Rg)
Previous display: g_felt = (v²/Rg) — the denominator "Rg" reads ambiguously as either R×g or R/g depending on reader. Fixed: All formula displays now use unambiguous dot notation: v²/(R·g). Also added University of Gothenburg / Liseberg Physics (2005) citation on clothoid loop shape.
✓ PASS-2 FINDING 5 — FIXED: Safety table labeled 5g as "SAFE"
Previous table row said "Typical coaster max = 5.0g → SAFE." This conflicts with the g-force thresholds now correctly implemented. Fixed: Row now reads "Design ceiling (Tony Wayne): 5.0g → DESIGN LIMIT" with a separate row clarifying that typical operating range is 3–4g, citing Wikipedia and CoasterForce.
✓ PASS-2 FINDING 6 — FIXED: Track Builder loop labeled as generic "LOOP"
The Track Builder uses a single radius for both top and bottom of the loop segment (circular approximation). The Loop Designer tab (Module 4) correctly uses different R_bottom and R_top per irregular/clothoid design. Fixed: Segment now labeled "LOOP (CIRC)" with a note directing users to Module 4 for irregular loop design.
