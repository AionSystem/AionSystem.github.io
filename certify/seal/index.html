FROZEN DECLARATION — FROZEN-2.0

      
This tool implements the FROZEN-2.0 stamp function. The SHA-256 seal is computed locally in your browser using SubtleCrypto. The GitHub ledger entry is created automatically via the AION backend — no GitHub account required. FROZEN-2.0 was rebuilt from first principles after FROZEN-1.0 was retired due to an incomplete dehiyot implementation. The defect is documented permanently in the ledger.

    

  

  

    
HOW THE AUTO LEDGER WORKS

    
When you generate a seal, two things happen simultaneously: (1) SHA-256 is computed locally in your browser — this never touches a server and is cryptographically final. (2) The AION backend is called to create a GitHub issue in the SOVEREIGN-TRACE-PROTOCOL repository automatically. The issue carries your seal hash, triple-time stamp, and template classification. You receive the issue URL immediately.

    
You do not need a GitHub account. You do not need to touch GitHub at all. The ledger entry is created for you. The SHA-256 seal is the truth — the GitHub issue is the permanent public witness.
